# Сокращения, обозначения и термины ПО НА NATALIA

## 1. Назначение документа

Настоящий документ содержит перечень сокращений, обозначений, английских идентификаторов и терминов, используемых в
документации по алгоритму штатного ПО НА NATALIA.

Документ предназначен для:

- единообразного использования терминов во всех алгоритмических документах;
- уменьшения неоднозначности в описании режимов, команд, событий и аварийных признаков;
- подготовки итоговой версии документации к переносу в LaTeX.

## 3. Термины уровня алгоритма

| Термин                           | Определение                                                                                                                                               | Примечание                                             |
|----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------|
| Режим                            | Верхнеуровневое состояние работы ПО                                                                                                                       | Например: `DUTY`, `OBSERVE`, `ALARM`                   |
| Подсостояние                     | Внутреннее состояние внутри режима                                                                                                                        | Например: `OBS_ACTIVE`, `TEST_SAVE`                    |
| Событие                          | Внешний или внутренний триггер обработки алгоритма                                                                                                        | Команда, тик `RTC`, авария и т.д.                      |
| Переход                          | Смена режима или подсостояния                                                                                                                             | Выполняется по событию и условию                       |
| Действие                         | Операция, выполняемая алгоритмом                                                                                                                          | Включение питания, запись MRAM и т.д.                  |
| Целевая операция                 | Основная длительная операция режима                                                                                                                       | Стирание, тестирование, наблюдение, вывод              |
| Безопасная конфигурация          | Состояние, в котором целевая активность прекращена, ПЭД отключена, питание NAND отключено, интерфейсы отключаемых узлов переведены в безопасное состояние | Используется в логике `ALARM` и `SHUTDOWN`             |
| Активный банк NAND               | Банк NAND, используемый в текущем режиме                                                                                                                  | `NAND1` или `NAND2`                                    |
| Сеанс наблюдений                 | Непрерывный цикл работы в режиме `OBSERVE`                                                                                                                | Имеет идентификатор `session_id`                       |
| Номер режима наблюдений          | Идентификатор внутренней конфигурации внутри одного сеанса наблюдений                                                                                     | Инкрементируется при `CMD_OBSERVE_CTRL`                |
| Формат НИ                        | Структурированный блок научной или служебной информации                                                                                                   | «События», «Счетчики», «Телеметрия» и т.д.             |
| Пакет НИ                         | Базовая единица хранения НИ в NAND                                                                                                                        | Пакет размером `2048` байт                             |
| Переполнение NAND                | Состояние, при котором активный банк больше не допускает продолжение записи НИ                                                                            | Приводит к автоматическому завершению `OBSERVE`        |
| Маска аварий                     | Битовое поле маски аварийных признаков                                                                                                                    | Хранится в MRAM                                        |
| Аварийный статус                 | Битовое поле активных аварийных признаков                                                                                                                 | `AlarmStatus`                                          |
| Замаскированный аварийный статус | Результат применения маски к аварийному статусу                                                                                                           | `MaskedAlarm`                                          |
| Валидная копия MRAM              | Копия данных MRAM, содержащая полный набор требуемых параметров и корректную CRC                                                                          | Используется при инициализации и восстановлении данных |
| Горячее резервирование MRAM      | Хранение одинаковых данных в двух копиях MRAM с взаимной проверкой и восстановлением                                                                      | Используется для конфигурации и служебных данных       |
| Согласованная запись             | Минимальный логически завершенный элемент записи, допускаемый к корректному завершению перед выходом из режима                                            | Для режима `OBSERVE` соответствует одному формату НИ   |

## 4. Обозначения верхнеуровневых режимов

| Обозначение | Наименование            | Значение                                    |
|-------------|-------------------------|---------------------------------------------|
| `INIT`      | Инициализация           | Инициализация после подачи питания          |
| `DUTY`      | Дежурный режим          | Базовый режим ожидания и маршрутизации      |
| `ERASE`     | Режим стирания NAND     | Стирание выбранного банка NAND              |
| `TEST`      | Режим тестирования NAND | Тестирование выбранного банка NAND          |
| `OBSERVE`   | Режим наблюдений        | Прием событий ПЭД, формирование и запись НИ |
| `DUMP`      | Режим вывода данных     | Вывод накопленных пакетов НИ                |
| `ALARM`     | Аварийный режим         | Перевод системы в безопасную конфигурацию   |
| `SHUTDOWN`  | Режим выключения        | Подготовка к снятию питания                 |

## 5. Обозначения подсостояний режимов

### 5.1 Подсостояния режима наблюдений

| Обозначение      | Наименование               | Значение                                    |
|------------------|----------------------------|---------------------------------------------|
| `OBS_ENTER`      | Подготовка входа           | Подготовка NAND, ПЭД и параметров режима    |
| `OBS_ACTIVE`     | Активное наблюдение        | Основной цикл режима наблюдений             |
| `OBS_EXIT_CMD`   | Завершение по команде      | Завершение по `CMD_DUTY` или `CMD_SHUTDOWN` |

### 10.1 Немаскируемые аварийные признаки

К немаскируемым аварийным признакам относятся:

- `ALARM_NAND_PS`;
- `ALARM_NAND_PR`;
- `ALARM_USB_PR`.

Для указанных признаков соответствующие биты `AlarmMask` должны быть всегда равны `1`.

## 13. Предпочтительные термины для использования в документации

| Предпочтительный термин   | Не рекомендуется            | Причина                                                           |
|---------------------------|-----------------------------|-------------------------------------------------------------------|
| режим наблюдений          | режим сбора                 | В документации принят термин «наблюдения»                         |
| телеметрическое сообщение | ответка / телеметрия наружу | Требуется точное техническое наименование                         |
| аварийный статус          | ошибка                      | Ошибка и авария не являются синонимами                            |
| команда управления        | пакет команды               | Требуется единообразное обозначение                               |
| команда телеметрии        | входная телеметрия          | Во избежание смешения с ТС                                        |
| пакет НИ                  | кусок данных                | Требуется точное соответствие формату                             |
| активный банк NAND        | текущая флешка              | Исключается разговорная терминология                              |
| безопасная конфигурация   | safe-state                  | Для основного текста предпочтителен русский термин                |
| интерфейс `PED_REG`       | шина ПЭД                    | Требуется использовать формальное обозначение интерфейса          |
| валидная копия MRAM       | рабочая флешка MRAM         | Требуется различать физический банк и логическое состояние данных |

## 14. Правила использования английских идентификаторов

В документации приняты следующие правила:

- для режимов и подсостояний используются английские идентификаторы в моноширинной записи;
- в поясняющем тексте используется русское наименование;
- для команд применяются английские символические имена;
- для аварий применяются обозначения, согласованные с протоколом и алгоритмической документацией;
- в итоговой LaTeX-версии английские идентификаторы сохраняются как формальные обозначения.
