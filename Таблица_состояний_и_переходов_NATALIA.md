# Таблица состояний и переходов ПО НА NATALIA

## 1. Назначение документа

Настоящий документ содержит формальное описание верхнеуровневого автомата состояний штатного ПО платы управления
гамма-спектрометра НА NATALIA.

Документ определяет:

- перечень верхнеуровневых состояний;
- перечень внешних и внутренних событий;
- условия выполнения переходов;
- действия, выполняемые при переходах;
- результирующее состояние после обработки события.

Документ используется совместно с:

- основным описанием алгоритма;
- документом обработки команд CAN;
- документами по отдельным режимам работы;
- документом допущений и открытых вопросов;
- диаграммами состояний Mermaid.

---

## 2. Общие правила использования

### 2.1 Формат описания перехода

Каждый переход описывается пятёркой:

**(текущее состояние, событие, условие, действие, следующее состояние)**

где:

- **текущее состояние** — режим, в котором находится ПО до обработки события;
- **событие** — внешняя команда, внутреннее событие или изменение контролируемого состояния;
- **условие** — дополнительное логическое ограничение;
- **действие** — набор операций, выполняемых ПО;
- **следующее состояние** — режим после завершения обработки события.

### 2.2 Недопустимые события

Если событие недопустимо в текущем состоянии:

- для КУ формируется ТС «Квитанция» с кодом ошибки;
- для КТ команда игнорируется;
- для прочих внутренних событий состояние не изменяется, если иное не указано явно.

### 2.3 Уровень детализации

Настоящая таблица описывает только верхнеуровневые переходы.  
Внутренняя логика отдельных режимов описывается в документах:

- `mode_duty.md`
- `mode_observe.md`
- `mode_erase.md`
- `mode_test.md`
- `mode_dump.md`
- `mode_alarm.md`
- `mode_shutdown.md`

### 2.4 Приоритет безопасности

Во всех режимах приоритет безопасного перевода аппаратуры в допустимое состояние выше приоритета выполнения целевых
операций.

## 3. Перечень верхнеуровневых состояний

| ID | Обозначение | Наименование            | Назначение                                  |
|----|-------------|-------------------------|---------------------------------------------|
| S0 | INIT        | Инициализация           | Стартовое состояние после подачи питания    |
| S1 | DUTY        | Дежурный режим          | Базовый режим ожидания и маршрутизации      |
| S2 | ERASE       | Режим стирания NAND     | Стирание выбранного банка NAND              |
| S3 | TEST        | Режим тестирования NAND | Тестирование выбранного банка NAND          |
| S4 | OBSERVE     | Режим наблюдений        | Приём событий ПЭД, формирование и запись НИ |
| S5 | DUMP        | Режим вывода данных     | Вывод накопленных пакетов НИ                |
| S6 | ALARM       | Аварийный режим         | Перевод системы в безопасную конфигурацию   |
| S7 | SHUTDOWN    | Режим выключения        | Подготовка НА к снятию питания              |

Примечание:  
Состояние `INIT` является алгоритмическим состоянием и используется для формального описания стартовой логики.

## 5. Общие правила приоритета событий

При одновременном возникновении нескольких событий использовать следующий логический приоритет обработки:

1. `E_MASKED_ALARM_SET`
2. `E_CMD_SHUTDOWN`
3. события штатного завершения длительных операций (`E_ERASE_DONE`, `E_TEST_DONE`, `E_DUMP_DONE`)
4. остальные допустимые КУ
5. допустимые КТ
6. `E_RTC_1HZ`
7. служебные события повторной передачи ТС
